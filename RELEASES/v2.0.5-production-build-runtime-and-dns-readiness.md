## What Changed
- Switched production build script to webpack mode to avoid Next 16 Turbopack EPERM runtime panic in this VPS environment:
  - `build`: `npm run build:info && next build --webpack`
- Updated deploy verification logging to prevent resolved env values leaking into qa artifacts:
  - Added no-output execution path for `docker compose ... config` in `scripts/verify-before-deploy.sh`.
- Verified deploy gate sequence on VPS:
  - `npm run verify:deploy` now passes end-to-end (including compose config/build) when run with real VPS permissions.
- Added DNS/TLS go-live runbook:
  - `docs/DNS_AND_TLS_GO_LIVE.md` with required record (`admin.youlya365.com -> VPS IPv4`), propagation checks, and strict TLS validation flow.
- Confirmed DNS now resolves to VPS IPv4, but TLS readiness still fails due invalid/expired certificate.
- Confirmed `certbot` is installed, while `nginx` service/binary is not present on this VPS (blocks `certbot --nginx` path as-is).
