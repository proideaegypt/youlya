## What Changed
- **Handoff Service** (`lib/services/handoff-service.ts`): Complete rewrite with full CRUD:
  - `createHandoffTicket` — creates/upserts open handoff, pauses AI
  - `assignHandoff` — assigns ticket to staff member
  - `resolveHandoff` — closes ticket, removes AI pause
  - `returnToAI` — returns conversation to AI, resolves any open handoff
  - `addNoteToHandoff` — appends internal notes
  - `isAIPausedForConversation` — checks DB or mock state
  - `getOpenHandoffs` — lists open/assigned tickets with store filter
- **AI Pause State** (`lib/services/conversation-flow-service.ts`): Added `setAIPaused`, `isAIPaused` with DB persistence via `conversation_state.ai_paused` column.
- **Mock Store** (`lib/adapters/supabase/mock-store.ts`): Added `aiPausedConversations` Set for test/dev parity.
- **Migration** (`supabase/migrations/20260504060000_handoff_center_and_ai_pause.sql`): Adds `notes` to `handoff_tickets`, `ai_paused` to `conversation_state`, and status check constraint.
- **Message-turn integration** (`lib/services/message-turn-service.ts`): Added `isAIPaused` check after kill-switch/handoff. If paused, returns static handoff reply with no AI commerce action.
- **Dashboard APIs** (`app/api/dashboard/handoff/`):
  - `GET /api/dashboard/handoff` — list with status/priority filters
  - `POST /api/dashboard/handoff/[id]/assign`
  - `POST /api/dashboard/handoff/[id]/return-to-ai`
  - `POST /api/dashboard/handoff/[id]/resolve`
  - `POST /api/dashboard/handoff/[id]/note`
- **Dashboard UI** (`app/dashboard/handoff/page.tsx`): Handoff Center with queue cards, priority/status filters, conversation preview, assign/return-to-AI/resolve buttons, internal notes panel.
- **Sidebar** (`lib/ui/dashboard-sidebar.tsx`): Added "التحويل البشري / Handoff Center" nav item.
