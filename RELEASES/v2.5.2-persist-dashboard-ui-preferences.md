# Release: v2.5.2 — persist-dashboard-ui-preferences

## Task
persist-dashboard-ui-preferences

## Release Type
patch

## What Changed
- Created centralized `lib/ui/preferences.ts` module with safe SSR-guarded localStorage/cookie helpers.
- Migrated storage keys to consistent naming:
  - `youlya.theme` (was `youlya-theme`)
  - `youlya.colorTheme` (was `youlya-brand`)
  - `youlya.language` (was `youlya_lang`)
  - `youlya.sidebarCollapsed` (was `youlya-sidebar-open`)
- Added pre-hydration script in `app/layout.tsx` to apply color theme and language direction before React paint, eliminating flash-of-default-state on reload.
- Added one-time localStorage key migration in pre-hydration script and theme provider.
- Updated `theme-provider.tsx` to use `storageKey="youlya.theme"`, disabled system theme for explicit control, and added cookie sync.
- Updated `theme-toggle.tsx` to write `youlya-theme` cookie on every change.
- Updated `color-theme.tsx` to use validated `ValidColor` type, write `youlya-color-theme` cookie, and apply `data-brand` immediately.
- Updated `dashboard-shell.tsx` to persist language to localStorage + cookie, and fixed logout to use `supabase.auth.signOut()` instead of calling non-existent `/api/auth/logout`.
- Updated `dashboard-sidebar.tsx` to persist collapsed state to localStorage + cookie, and fixed fallback logout to use Supabase signOut.
- Added Playwright persistence test (`dashboard-preferences-persistence.spec.ts`) covering reload and logout/login cycles.
- Added `dashboard-preferences-persistence` project to `playwright.config.ts`.
