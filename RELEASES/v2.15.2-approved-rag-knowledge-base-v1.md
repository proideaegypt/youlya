approved-rag-knowledge-base-v1

## What Changed
- Added approved RAG schema foundation:
  - `knowledge_base` (published knowledge only for retrieval),
  - `knowledge_versions` (versioned publish history),
  - `knowledge_suggestions` (approval queue for learning suggestions).
- Implemented explicit approval/publish flow service with mock + Supabase paths:
  - review actions: `approve`, `reject`, `publish`,
  - publish creates `knowledge_base` entry + `knowledge_versions` v1 snapshot,
  - no automatic learning ingestion to production knowledge.
- Added store-scoped retrieval endpoint:
  - `POST /api/ai/rag/retrieve`
  - input requires `store_id`,
  - returns approved snippets only + approved-only prompt context.
- Added approved-only prompt builder:
  - `buildApprovedKnowledgePrompt(...)` includes approved snippet text only.
- Added dashboard moderation API + UI:
  - `GET/POST /api/dashboard/knowledge-base`
  - new page `/dashboard/knowledge-base`
  - visibility for pending/approved/rejected/published with approve/reject/publish actions.
- Added unit tests for retrieval/prompt/publish workflow behavior.
