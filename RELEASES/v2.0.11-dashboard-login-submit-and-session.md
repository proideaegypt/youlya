## What Changed
- Added SSR-compatible Supabase browser auth client at `lib/supabase/browser.ts` using `@supabase/ssr` `createBrowserClient` with public env only.
- Updated login submit flow in `app/login/login-form.tsx` to use SSR-compatible browser client, preserve loading/error UX, and use `router.replace("/dashboard") + router.refresh()` on success.
- Added visible connection-status indicator in login form showing whether public Supabase env is configured (without exposing values).
- Hardened dashboard layout guard in `app/dashboard/layout.tsx` to validate real Supabase user via server-side `createServerClient(...).auth.getUser()` instead of cookie-name heuristic only.
- Added focused unit tests:
  - `tests/unit/supabase-browser-client.test.ts`
  - `tests/unit/root-page.test.ts`
- Added baseline and final QA artifacts for this task under `qa-artifacts/tasks/2026-05-01/fix-dashboard-login-submit-and-session/`.
