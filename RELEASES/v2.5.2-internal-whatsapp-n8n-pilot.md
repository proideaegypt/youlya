## What Changed
- Fixed Arabic digit (٠-٩) selection in intent-detector, select-product-service, and message-turn-service.
- Added `normalizeArabicDigits` helper to convert Arabic-Indic digits to ASCII before `Number()` parsing.
- Added mock-state fallback in `product-mapping-service.ts` so product selection works when production DB schema mismatches.
- Created forward-only migration `20260502050000_fix_conversation_id_text.sql` to ALTER `conversation_id` from UUID to TEXT in `processed_messages`, `human_handoffs`, `dead_letter_log`, `handoff_tickets`, and `ai_tool_calls`.
- Updated existing migration files to use `TEXT` for `conversation_id` so future installs get correct schema.
