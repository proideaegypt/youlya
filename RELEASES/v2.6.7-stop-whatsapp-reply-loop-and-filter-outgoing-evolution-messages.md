## What Changed
- Added a `Guard Inbound Customer Message` code node immediately after the webhook so outgoing Evolution messages with `fromMe=true` are dropped before any turn logic runs.
- Hardened `Normalize Message` to refuse outgoing payloads defensively and to normalize `remote_jid`, `send_number`, `text`, and `provider_message_id` from the actual Evolution webhook payload.
- Fixed `Prepare Reply` to compute `number` before `shouldSend`, so the reply gate only passes when a real recipient and reply both exist.
- Updated `Send Text` to use the normalized recipient fallback chain and verified the live workflow with synthetic webhook requests only.
