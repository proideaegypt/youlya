## What Changed
- Created `supabase/migrations/20260502180000_shopify_product_variant_sync.sql` — safe forward-only migration creating `products` and `product_variants` tables with indexes and RLS policies, plus augmenting `last_product_recommendations` with missing columns.
- Created `lib/adapters/shopify/shopify-product-sync-adapter.ts` — Shopify Admin GraphQL product fetcher with pagination, rate limit handling, size/color extraction from selectedOptions, SKU missing detection.
- Created `lib/adapters/supabase/product-sync-repository.ts` — Supabase upsert repository for products and variants with `available_for_ai` logic and `code_missing` flag.
- Created `lib/services/shopify-product-sync-service.ts` — sync orchestration service supporting `full` and `inventory` modes, dry-run, and safe error handling.
- Created `app/api/internal/shopify/sync-products/route.ts` — internal authenticated POST endpoint for n8n daily workflow.
- Created `scripts/sync-shopify-products.mjs` — manual CLI script with `--mode`, `--dry-run`, `--store-id` flags.
- Updated `lib/services/product-search-service.ts` — now queries Supabase `products` + `product_variants` cache first, falls back to mock catalog.
- Created `n8n/workflows/youlya-daily-shopify-product-sync.json` — sanitized n8n daily workflow (inactive by default) calling internal sync endpoint.
- Updated `scripts/validate-n8n-workflows.mjs` — now validates both WhatsApp canonical workflow and Shopify daily sync workflow.
- Created `docs/SHOPIFY_PRODUCT_SYNC_RUNBOOK.md` — comprehensive runbook.
- Added package scripts: `shopify:sync`, `shopify:sync:inventory`, `shopify:sync:dry-run`.
- No secrets were printed or committed. No live Shopify data mutated.
