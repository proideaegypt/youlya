## Release Type

## What Changed
- Fixed production n8n webhook registration: `POST https://ai.youlya365.com/webhook/youlya-whatsapp` was returning 404 because the webhook node lacked a `webhookId`, causing n8n to register it under a workflow-ID-prefixed path (`joqfame4HXG775JO/webhook/youlya-whatsapp`) instead of the expected simple path (`youlya-whatsapp`).
- Added `webhookId` UUID to the live n8n workflow webhook node, updated `typeVersion` from `2` to `2.1`, and added `"options": {}`.
- Deactivated and reactivated the workflow to force n8n to re-register the webhook with the correct path.
- Updated canonical repo workflow `n8n/workflows/youlya-whatsapp-main.json` with the same `webhookId`, `typeVersion: 2.1`, and `options: {}` to prevent future imports from hitting the same issue.
- Saved pre-edit backup to `qa-artifacts/tasks/2026-05-03/register-youlya-n8n-production-webhook/backup-youlya-whatsapp-main.json`.
- Verified production webhook now returns HTTP 200 `{"message":"Workflow was started"}` and n8n executions are created.
