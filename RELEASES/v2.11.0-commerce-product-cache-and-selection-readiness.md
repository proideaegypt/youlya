## What Changed
- `shopify:assert-readonly` — PASS: confirmed no forbidden Shopify mutation/write patterns in product sync path.
- Improved Arabic selection parser in `lib/services/select-product-service.ts`:
  - Added Arabic ordinal support: `اول`, `اولى`, `الاول`, `التاني`, `التانية`, `الثاني`, `الثالث`, `الرابع`, `الخامس`, etc.
  - Added Arabic size name support: `اكس لارج`, `لارج`, `ميديم`, `سمول`, `كبير`, `صغير`, etc.
  - Added `نمرة` (number) pattern support alongside `رقم`.
  - All Arabic digits (`٠-٩`) normalized before parsing.
- Added comprehensive search and selection validation tests in `tests/unit/select-product.test.ts` (13 tests total):
  - Arabic ordinal selection (`اول واحد`, `التاني`, `الخامس`)
  - Arabic digit selection (`نمرة ١`)
  - Arabic size names (`اكس لارج`, `لارج`, `ميديم`)
  - Product search validation: max 10 results, `shopifyProductId` present, `shopifyVariantId` present internally, OOS correctly marked unavailable.
  - Multi-item selection, mapping expiry, and out-of-stock blocking.
- Verified `lib/services/product-mapping-service.ts` persists recommendations to `last_product_recommendations` and resolves variants from DB (not LLM memory).
- Verified `lib/services/product-search-service.ts` searches Supabase cache first with `ilike` title match, fallback to mock catalog in dev/test.
