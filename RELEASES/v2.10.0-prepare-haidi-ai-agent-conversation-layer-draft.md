## What Changed
- Added `docs/HAIDI_AI_SALES_AGENT_PROMPT.md` — production-ready system prompt for Haidi AI Sales Agent
- Added `docs/HAIDI_MEMORY_DESIGN.md` — safe memory boundaries and data flow for Haidi layer
- Added `lib/services/haidi-output-validator.ts` — validates Haidi JSON output, blocks unsafe order claims, internal ID exposure, unverified price/stock claims, falls back to app reply
- Added `lib/services/haidi-context-builder.ts` — builds safe commerce facts context for Haidi consumption (product facts, cart summary, reply goal, style instructions)
- Extended `lib/types/messages.ts` — added optional `haidi_context` to `MessageTurnResponse`
- Updated `lib/services/message-turn-service.ts` — populates `haidi_context` for all major intents (PRODUCT_SEARCH, SELECT_PRODUCT, CONFIRM_ORDER, handoff, unclear, fallback, kill switch, angry tone)
- Added `n8n/workflows/youlya-whatsapp-main-haidi-draft.json` — draft workflow with Haidi layer (active=false), includes Haidi AI Sales Agent placeholder node and Validate Haidi Output node
- Added `n8n/workflows/HAIDI_AGENT_WORKFLOW_PATCH_PLAN.md` — exact specification for adding Haidi layer to active workflow later
- Added `tests/unit/haidi-agent.test.ts` — 21 tests covering validator and context builder
- Updated `scripts/validate-n8n-workflows.mjs` — validates Haidi draft workflow presence, active=false, no Shopify direct nodes, no hardcoded secrets
