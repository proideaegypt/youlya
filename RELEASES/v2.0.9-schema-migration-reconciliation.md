## What Changed
- Added mandatory automated production SQL safety gate script:

  - package script: `npm run check:migration:safe -- <migration.sql>`
- Safety gate enforces hard-fail on forbidden patterns:

- Confirmed current production schema has no missing app-critical tables.
- Ran safety gate against reconciliation migration and confirmed expected fail on forbidden `cascade`, preventing any new unsafe production SQL apply in this pass.
- Added process documentation:
  - `docs/SCHEMA_MIGRATION_RECONCILIATION.md`
  - includes inventory, reconcile, safety-check workflow and no-destructive-SQL policy.
