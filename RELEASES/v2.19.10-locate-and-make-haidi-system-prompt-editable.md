## What Changed
- Identified the live Haidi prompt source in the active n8n workflow and confirmed it was inline inside the `Build Haidi Prompt` Code node.
- Added a repo-backed prompt-state service with repo fallback, draft/publish state, rollback support, and lab-gated publishing.
- Added `GET /api/dashboard/haidi/prompt` plus draft/publish/rollback/test POST handling under dashboard auth.
- Updated `/dashboard/haidi/settings` to show prompt source/version, edit the prompt draft, test it through Haidi Lab, publish only after a passing run, and roll back to the previous version.
- Added the prompt state migration and prompt-service unit tests.
- Patched the active `Youlya WhatsApp Main` n8n workflow so the prompt node now consumes `haidi_context.prompt.currentPrompt` from the app instead of a hardcoded inline system prompt.
- Updated the repo prompt doc and workflow patch plan to reflect repo-default fallback plus dashboard override behavior.
