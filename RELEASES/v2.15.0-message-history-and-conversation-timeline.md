## What Changed
- Added forward-only Supabase migration `20260504050000_message_history_and_timeline.sql`:
  - Extended `conversations` with `ai_paused`, `assigned_to`, `last_message_at`, `external_conversation_id`
  - Extended `messages` with `direction`, `transcript`, `image_caption`, `ai_agent_draft`, `final_reply`, `status`, `n8n_execution_id`, `evolution_message_id`, `error_code`, `text`
  - Created `conversation_events` table for tool calls, handoffs, errors, cart updates
  - Added indexes for dashboard query performance
- Created `lib/services/message-history-service.ts` with:
  - `logInboundMessage()`, `logOutboundMessage()`, `logSystemEvent()`
  - `getConversationTimeline()`, `listConversations()`
  - `maskCustomerIdentifier()` and secret stripping
- Integrated logging into `app/api/internal/messages/turn/route.ts` and `app/api/webhooks/evolution/route.ts`
- Updated dashboard API:
  - `GET /api/dashboard/conversations` — list with masked customer IDs and status
  - `GET /api/dashboard/conversations/[id]` — detail with timeline
  - `GET /api/dashboard/conversations/[id]/timeline` — dedicated timeline endpoint
- Updated `app/dashboard/inbox/page.tsx` with conversation list, status badges, and full message/event timeline
- Added `tests/unit/message-history-service.test.ts` covering persistence, PII masking, secret stripping, timeline sorting
- Updated `lib/adapters/supabase/mock-store.ts` to support mock history data
