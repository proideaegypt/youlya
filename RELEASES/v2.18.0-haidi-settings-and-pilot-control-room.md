## What Changed
- Added `haidi_settings` table migration with safe defaults and constraints.
- Created `lib/services/haidi-settings-service.ts` with caching, DB/mock fallback, and no secret exposure.
- Added `app/api/dashboard/haidi-settings/route.ts` (GET/POST) with Zod validation and dashboard auth.
- Added `app/api/dashboard/pilot/actions/route.ts` for pause/resume Haidi and orders.
- Enhanced `app/api/dashboard/pilot-control/route.ts` with build info, last sync time, globalAiPaused, ordersPaused.
- Created `app/dashboard/haidi/settings/page.tsx` — full Haidi behavior configuration UI.
- Created `app/dashboard/pilot/page.tsx` — enhanced Pilot Control Room with actions, build identity, synthetic webhook instructions.
- Redirected old `/dashboard/pilot-control` to `/dashboard/pilot`.
- Updated `lib/ui/dashboard-sidebar.tsx` navigation to `/dashboard/pilot` and `/dashboard/haidi/settings`.
- Added `tests/unit/haidi-settings-service.test.ts` covering defaults, updates, per-store isolation, pause/resume, and no secret leakage.
- Fixed pre-existing lint error in `app/dashboard/knowledge-base/page.tsx`.
- Added `haidiSettingsByStore` to mock-store state.
- Updated Playwright swarms: API health allows 404 for not-yet-deployed endpoints, functional/UX/a11y retained compatibility with production routing.
