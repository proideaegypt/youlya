## Release Type

## What Changed
- Added Haidi Lab scenario data model:
  - `haidi_lab_scenarios` for scenario CRUD,
  - `haidi_lab_runs` for actual run output + scoring/mismatches.
- Added Haidi Lab service:
  - scenario CRUD (list/create/update/delete),
  - safe run execution through `runMessageTurn` with `testMode: true`,
  - expected-vs-actual scoring (intent/tone/must_include/must_not_include),
  - reply masking for PII-like data patterns.
- Added learning suggestion creation from lab runs into `knowledge_suggestions` as `pending`.
- Added dashboard APIs:
  - `GET/POST /api/dashboard/haidi/lab`
  - `PUT/DELETE /api/dashboard/haidi/lab/[id]`
  - `POST/PUT /api/dashboard/haidi/lab/run` (run + create learning suggestion)
  - `GET/POST /api/dashboard/haidi/learning` (approval queue actions + publish placeholder)
- Added dashboard pages:
  - `/dashboard/haidi/lab`
  - `/dashboard/haidi/learning`
- Added sidebar navigation entries for Haidi Lab and Haidi Learning.
- Added targeted unit coverage for CRUD, run safety, scoring, approval queue, and non-exposure checks.
