## What Changed
- Added a safe `docs/pilot-sprint/` playbook set that reflects the current live Youlya architecture instead of the original zero-state sprint assumptions.
- Sanitized operational language so host/IP/secret references use placeholders and the public app domains remain the only hardcoded hostnames.
- Updated the playbook to mark the current state correctly: live dashboard, products dashboard, Shopify sync proof, n8n runtime env, and the existing WhatsApp loop guard.
- Clarified the architecture split so Haidi remains a conversation/tone layer, the app owns commerce truth and safety, n8n orchestrates only, Shopify stays the source of truth, and Supabase remains the operational cache.
- Added and hardened `scripts/smoke-test-daily.sh` and `scripts/run-pilot-scenarios.sh` so they run synthetic checks, avoid secret printing, and produce artifacts without creating live orders.
- Added `tests/scenarios/pilot-day-5.jsonl` with 20 safe pilot cases covering product search, select product, address collection, confirmation, duplicate handling, voice, image, and handoff behavior.
- Updated the pilot scenario harness so it uses the internal message turn API safely with per-run idempotency keys and stateful conversation threads where needed.
- Added the emergency rollback note and the current next-action ordering for the pilot sprint.
- Verified the safety path with `npm run smoke:daily`, `npm run pilot:scenarios`, `npm run validate:scenarios`, `npm run scan:secrets`, `npm run typecheck`, `npm run lint`, `npm test`, and a clean `npm run build` after clearing stale `.next` artifacts.
